git tag 1.0 (commit id) creates a new tag with the name/identifier 1.0, the commit id specifies a certain commit to
git reset (filename) takes the current branch and resets it to point somewhere else
git stash list lists all the modifications that have been stashed away by this command
git stash pop removes a stashed state from the stash list and puts it back in the current working tree state
git rm -f forces a removal of files from the staging area and working directory
git rebase -i HEAD~ 3 allows you to interactively choose how each of the last 3 commits will be modified during rebase operations of the current head
